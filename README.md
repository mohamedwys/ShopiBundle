# **Shopify Product Bundler App**

## **Getting Started**

This guide helps you set up the repo and customize it for features like registering webhooks and adding routes.

### **Installation**

1. Run `npm i` to install dependencies.

- If you encounter any issues, feel free to open an issue.
- ********Important:******** Do not delete the `shopify.app.toml` file. It's needed for Shopify CLI 3.0.

2. Create a new app (Public or Custom) from your [Shopify Partner Dashboard](https://partners.shopify.com).

- For now, set the App URL to `https://localhost`.

3. Build your `.env` file based on `.env.example`.

- `SHOPIFY_API_KEY`: App API key.
- `SHOPIFY_API_SECRET`: App secret.
- `SHOPIFY_API_SCOPES`: Scopes required by your Shopify app. A list of access scopes can be found [here](https:/shopify.dev/api/usage/access-scopes)
- `SHOPIFY_APP_URL`: URL generated from Ngrok.
- `SHOPIFY_API_VERSION`: Pre-filled to the latest version. All the calls in the repo are based off this APIversion so if you're downgrading please refer to the official docs instead. The repo is always kept up to datewith the newest practices so you can rely on the basic repo to almost always work without depriciation errorspopping up.
- `DATABASE_URL`: Database connection URL. Since we're using Prisma ORM with this repo, it supports SQL and noSQLdatabases. Read more about it [here](https://www.prisma.io/stack)
- `ENCRYPTION_STRING`: String to use for Cryption for encrypting sessions token. Add a random salt (or a random string of letters and numbers) and save it. If you loose the string you cannot decrypt your sessions and must be kept safely.

### **NPM Scripts**

- `dev`: Run in dev mode
- `build`: Build for production
- `start`: Start in production mode. Requires `npm run build` before starting.
- `pretty`: Run `prettier` on the entire project.
- `update`: Force updates all packages to latest version and requires you to manually run `npm i --force` after.Not recommended if you don't know what you're doing.
- `update:url`: Use `@shopify/cli-kit` to update URLs to your Shopify partner dashboard. Requires a proper setupof `.env` file.
- `ngrok:auth`: Replace `<auth-token-goes-here>` with your ngrok token and run it to activate ngrok.
- `ngrok`: Starts ngrok on port 3000.
- `shopify`: Allows you to accesss Shopify CLI commands. Try `npm run shopify help` for more info.
- `s:e:create`: Create a Shopify extension.
- `s:e:deploy`: Deploy extension to Shopify.
- `pg:create`: Create a new folder `database` and init a PostgreSQL instance. Requires you to have postgresinstalled.
- Run `brew install postgresql`
- `pg:start`: Start a PostgreSQL instance on `database`. Requires you to run `npm run pg:create` before you can dothis.
- `pg:stop`: Stop PostgreSQL server.
- `prisma`: Generic command to access `prisma` commands.
- `prisma:push`: Push `schema.prisma` to your `DATABASE_URL` database.
- `prisma:pull`: Pull database schema from `DATABASE_URL` database and generates a `schema.prisma` file.
- `prepare`: Reserved script to generate `@prisma/client`.

### **Setup Partner Dashboard**

1. Run npm run ngrok to get a subdomain URL.
2. Update SHOPIFY_APP_URL in your .env file.
3. Add these URLs in the Shopify Partner Dashboard > App Setup:
- App URL: https://<your-url>
- Allowed Redirection URLs:
  - https://<your-url>/api/auth/callback
  - https://<your-url>/api/auth/tokens

### **Running App**

- If it's your first time connecting to said database, run `npx prisma db push` to get your database working.
- Run `npm run dev`, your database and ngrok.
- Install the app by heading over to `https://ngrokurl.io/api/auth?shop=mystorename.myshopify.com`.

### **Features**

- Bundle Creation and Management: Store owners can create and manage product bundles by selecting multiple products, setting a discount percentage, and assigning a unique name. They can also edit or delete existing bundles as needed.

- Automatic Bundle Generation: Store owners can set up rules to automatically generate product bundles based on criteria such as product tags, collections, or price range. They can easily configure and adjust these rules using an intuitive interface.

- Bundle Display and Pricing: The Shopify app prominently displays bundled products and the total discounted price on the product page, cart, and checkout. The components of the bundle are visually distinguished, and the original price and discounted price are clearly shown.

- Analytics and Reporting: The Shopify app provides comprehensive analytics to store owners, including metrics such as total sales generated by bundles, the popularity of different bundles, and individual bundle performance. The analytics dashboard is user-friendly and easy to understand.

- Responsive User Interface: The Shopify app has a responsive user interface that works seamlessly on both desktop and mobile devices. It utilizes React JS for the frontend and NextJS for server-rendered pages, ensuring a smooth and consistent experience.

- Multi-language Support and Localization: The Shopify app supports multiple languages, allowing store owners to provide a seamless experience for customers in different languages. It implements localization features to display the app's user interface and on-page elements in the preferred language of the store owner, and facilitates easy translation for additional languages as needed.
